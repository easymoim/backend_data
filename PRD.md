# 🧭 약속 비서 서비스 MVP 기획서

## 💡 1. 서비스 개요
**서비스명:** (예: 모여요 / 캘픽 / 더더더더)  
**한 줄 설명:** 빠르고 간편하게 모두의 시간을 모아 약속을 정하고, 중간 지점의 장소를 추천해주는 약속 비서 서비스  
**핵심 가치:**  
- 시간 조율 자동화  
- 중간지점 기반 최적 장소 추천  
- 모임 목적에 맞는 장소 추천 + 예약 연동 가능  

---

## 🎯 2. 주요 타깃
| 구분 | 설명 |
|------|------|
| 개인 사용자 | 친구, 연인, 동창 모임 등 사적인 약속 |
| 직장인 | 회식, 팀 미팅 등 업무 관련 모임 |
| 가족 단위 | 주말 외식, 행사 모임 등 가족 단위 일정 조율 |

## 🚀 3. 핵심 기능 (MVP)
| 기능명 | 설명 | 우선순위 |
|--------|------|----------|
| 시간 조율 | 참가자 가능한 시간 입력 → 겹치는 시간 자동 계산 | ⭐⭐⭐⭐⭐ |
| 중간지점 계산 | 참가자 위치 기반 공평한 중심점 계산 | ⭐⭐⭐⭐ |
| 장소 추천 | 중심지 주변 음식점/카페 등 추천 | ⭐⭐⭐⭐ |
| 약속 확정/공유 | 시간, 장소 확정 후 공유 링크 생성 | ⭐⭐⭐⭐ |
| 로그인/사용자 관리 | Google / Kakao OAuth 로그인 | ⭐⭐⭐ |
| 예약 연동 (확장) | 네이버예약, 카카오맵 예약 연동 | ⭐⭐ |

---

## 🧭 4. 사용자 시나리오
1. 로그인 → “새 약속 만들기” 클릭  
2. 제목, 날짜 후보 입력 → 초대 링크 공유  
3. 참가자 각자 가능한 시간과 출발지 입력  
4. 시스템이 겹치는 시간 + 중간 위치 계산  
5. 주변 장소 추천 → 선택 → 약속 확정  
6. 공유 링크 발송 (카톡, 문자 등)  

---

## 🏗️ 5. 기술 스택
**Frontend:** Next.js, React, TailwindCSS, Shadcn UI, Zustand, Kakao Map SDK  
**Backend:** FastAPI, Supabase(PostgreSQL), SQLAlchemy, Alembic, Redis, Celery, 외부 API(Kakao/Naver/Google)  
**Infra:** AWS (EC2, RDS, S3), Docker, GitHub Actions  

---

## 🧩 6. 데이터 모델 (핵심)
| 테이블 | 설명 |
|--------|------|
| users | 사용자 계정, OAuth 정보 |
| meetings | 약속 메인 정보 |
| participants | 참가자 목록 |
| availabilities | 참가자별 가능한 시간 |
| locations | 참가자 출발지 좌표 |
| recommendations | 추천 장소 결과 |
| places | 장소 API 캐싱용 데이터 |

---

## 🧠 7. 추천 알고리즘 (초기 버전)
1. 참가자 출발지 → 위도/경도 변환  
2. 중심점 계산 (평균 or 최소거리합)  
3. 중심점 반경 내 장소 검색  
4. 평점/리뷰/카테고리 필터링  
5. 상위 N개 추천 반환  

---

## 📊 8. 팀별 역할 (백엔드 1 + 데이터 1 기준)

### 백엔드
- FastAPI 환경 세팅, DB 모델 작성  
- CRUD API 개발 (`/meetings`, `/participants`, `/availabilities`)  
- 추천 API 연동 (`/recommendations`)  
- 공유 링크 생성 및 배포

### 데이터
- 장소 데이터 수집/정제 (Kakao, Naver, Google)  
- 중간지점 계산, 추천 알고리즘 개발  
- 추천 결과 캐싱 및 백엔드 연동
